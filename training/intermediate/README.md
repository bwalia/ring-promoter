# Intermediate path (~2 hours)

For app and platform engineers who know the basics.

1. The three **deployers** — kubectl ([hello-world](../sample-apps/hello-world)),
   github ([operator](../sample-apps/operator)), k8sjob ([image-proc](../sample-apps/image-proc)).
2. **Health strategies** — version by JSON field vs response header vs
   status-only ([sample-apps matrix](../sample-apps/README.md#feature-coverage-matrix)).
3. **Auto-promote** chains and when to stop them.
4. **Application groups** — Lab 07 with [shopping-cart](../sample-apps/shopping-cart).
5. Lab 06 — fix a failed deployment (version mismatch → auto-rollback).

**You'll be able to:** pick the right deployer + health strategy per app, wire
auto-promote, and manage multi-service apps as a group. Next:
[advanced](../advanced).
