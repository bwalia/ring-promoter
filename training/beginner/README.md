# Beginner path (~1 hour)

New to continuous delivery or GitOps? Start here.

1. Read [getting-started](../getting-started) — vocabulary and cluster access.
2. Concepts (15 min): rings, seed, promote, rollback, health checks — see
   [docs/architecture](../docs/architecture.md).
3. Do [Lab 01 — First deployment](../labs/lab-01-first-deployment.md).
4. Do Labs 02–03: promote a version int → test → acc (hello-world, kuard).
5. Do Lab 05: rollback.

**You'll be able to:** seed a version, promote it ring by ring, read a health
check, and roll back. Next: the [intermediate path](../intermediate).
